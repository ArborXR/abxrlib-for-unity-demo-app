PICO OpenXR (required for Pico headset builds)
==============================================

This project depends on the PICO **OpenXR** package (com.unity.xr.openxr.picoxr)
via Packages/manifest.json. That is **not** the same as the root package inside
"PICO Unity Integration SDK" (that root is com.unity.xr.picoxr — used by many
PICO projects without OpenXR).

This demo is **OpenXR-based** (XR Plug-in Management uses the OpenXR loader and
PICO OpenXR features). You still need the **OpenXR Integration SDK** contents
on disk, but they can live **inside** your existing PICO Unity Integration SDK
folder so everything matches your other projects’ layout.

Why you do not see vendor files on GitHub
-----------------------------------------
PICO SDKs are large and licensed; we do not commit them. Install locally on each
machine that builds for Pico.

Where to put files on disk (current layout)
-------------------------------------------
Relative to this Unity project root, the manifest points at:

  ../PICO Unity Integration SDK-3.2.0-20250707/PicoOpenXRPlugin

Example:

  .../arborxr/
    abxrlib-for-unity-demo-app/              <- Unity project (this repo)
      Assets/
      Packages/
      ...
    PICO Unity Integration SDK-3.2.0-20250707/   <- same folder you use elsewhere
      package.json                             <- com.unity.xr.picoxr (other projects)
      Runtime/
      ...
      PicoOpenXRPlugin/                        <- OpenXR plugin for *this* project
        package.json                           <- com.unity.xr.openxr.picoxr

Create PicoOpenXRPlugin by extracting PICO’s **Unity OpenXR Integration** ZIP
so that package.json (name com.unity.xr.openxr.picoxr) sits directly inside
PicoOpenXRPlugin/. If you already have that tree elsewhere, move or symlink it
to that path.

In Unity: Project > Packages > "PICO OpenXR Plugin" (embedded local package).

Setup checklist
---------------
1. OpenXR ZIP from PICO Developer Resources (Unity / OpenXR integration):
   https://developer.picoxr.com/resources/

2. Place the OpenXR package at:
   ../PICO Unity Integration SDK-3.2.0-20250707/PicoOpenXRPlugin
   (folder name may differ; then adjust manifest.json to match.)

3. Open the Unity project so Package Manager resolves the package.

4. Edit > Project Settings > XR Plug-in Management > Android: OpenXR enabled;
   PICO Support / profiles match your device.

If you see a flat black panel on device, the OpenXR path is wrong or missing, or
the Integration SDK version does not match what PICO documents for your Unity
version.
