PICO OpenXR (required for Pico headset builds)
==============================================

This project uses the PICO **OpenXR** package (com.unity.xr.openxr.picoxr) from a
**local folder** in manifest.json. PICO distributes it as the "Unity OpenXR
Integration" SDK ZIP; it is **not** the same package as "PICO Unity Integration
SDK" (com.unity.xr.picoxr at that ZIP’s root).

com.unity.xr.openxr (Unity) is on the Unity registry; **com.unity.xr.openxr.picoxr**
(PICO’s OpenXR features) comes from PICO only — use the extracted ZIP path below
if you do not have another approved way to supply it.

Why vendor files are not in git
-------------------------------
SDKs are large and PICO’s license restricts redistribution. Install locally per
machine (see LICENSE in the SDK).

Layout next to this Unity project
---------------------------------
Extract PICO’s OpenXR Integration ZIP so it sits **next to** this project folder
(sibling of the folder that contains Assets and Packages):

  .../arborxr/
    abxrlib-for-unity-demo-app/                    <- Unity project (this repo)
      Packages/
        manifest.json  ->  file:../../Unity OpenXR IntegrationSDK-1.4.0-20250407
      ...
    Unity OpenXR IntegrationSDK-1.4.0-20250407/    <- OpenXR package root
      package.json                                 <- com.unity.xr.openxr.picoxr
      Runtime/
      Editor/
      ...

Unity resolves `file:` paths in manifest.json **relative to the Packages folder**,
not the project root — so use `../../` to go up to the parent of the project, then
into the OpenXR folder. (`../` would incorrectly look inside the project folder.)

If your extracted folder name differs, edit Packages/manifest.json to match.

Optional (other PICO projects): you may also keep
`PICO Unity Integration SDK-3.2.0-20250707/` beside the project; this demo does
**not** reference that path for OpenXR.

Setup
-----
1. Download: PICO Developer Resources — Unity / OpenXR integration
   https://developer.picoxr.com/resources/

2. Unzip so `../../Unity OpenXR IntegrationSDK-1.4.0-20250407/package.json` exists
   relative to Packages/ (sibling of project folder; or change manifest to your folder name).

3. Open the Unity project; Package Manager should show "PICO OpenXR Plugin"
   under Packages (embedded).

4. Edit > Project Settings > XR Plug-in Management > Android: OpenXR enabled;
   PICO Support and device profiles as in PICO docs.

If the headset shows a flat black panel, the local package path is wrong or the
OpenXR ZIP version is incompatible with your Unity Editor version — check PICO’s
documentation for your Unity/OpenXR versions.

Unity 6 Android — Game Activity (project validation)
------------------------------------------------------
Unity 6 expects **Application Entry Point = Game Activity** for Android (OpenXR’s
Meta Quest Support check also flags legacy Activity). This repo sets that in
ProjectSettings (androidApplicationEntry = GameActivity). In the Editor: **Edit >
Project Settings > Player > Android > Other Settings > Application Entry Point >
Game Activity**.

PICO OpenXR ZIP — compiler / import warnings
--------------------------------------------
CS0108, CS0168, CS0414, CS1998, empty animation clips, and “GetDeviceLayoutName”
messages under `Unity OpenXR IntegrationSDK-...` are **PICO’s sources**, not this
demo. Do not hand-edit that tree (updates overwrite it; license is PICO’s). They
are safe to ignore for local development, or clear when PICO ships a newer SDK.

Optional: if you **only** target Pico, you can turn off **Meta Quest Support**
under **Project Settings > XR Plug-in Management > OpenXR > Android** to reduce
Meta-specific checks — Game Activity is still the right default for Unity 6.

OpenXR build fails: “Only the PICO Touch Interaction Profile is supported”
---------------------------------------------------------------------------
When **PICO Support** is enabled for Android, PICO’s OpenXR validation rejects
any **non-PICO** interaction profile (e.g. **Oculus Touch Controller Profile**).
In **Project Settings > XR Plug-in Management > OpenXR > Android**, disable
**Oculus Touch Controller Profile** (and keep a PICO profile such as PICO4 /
Neo3 enabled). **Meta Quest Support** should be off for Pico-only builds.

**appID is not configured** — from PICO’s Platform preprocessor when
`Assets/Resources/PXR_PlatformSetting.asset` has an empty **appID** (warning only
unless PICO changes it). Set a PICO developer **app ID** there if you use PICO
Platform services that require it.
